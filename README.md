# REST API для электронной библиотеки

## Описание проекта

Система управления электронной библиотекой на Django REST Framework.

## Технологии

- Python 3.13
- Django 5.0.1
- Django REST Framework 3.14.0
- SQLite

## Установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Примените миграции:
```bash
python manage.py migrate
```

3. Создайте администратора:
```bash
python manage.py createsuperuser
```

4. Запустите сервер:
```bash
python manage.py runserver
```

## API Endpoints

### Авторы

- `GET /api/authors/` - Список всех авторов
- `GET /api/authors/{id}/` - Информация об авторе
- `GET /api/authors/{id}/books/` - Книги автора

### Книги

- `GET /api/books/` - Список всех книг
- `GET /api/books/{id}/` - Информация о книге
- `POST /api/books/` - Создание книги (только админ)
- `PUT /api/books/{id}/` - Обновление книги (только админ)
- `PATCH /api/books/{id}/` - Частичное обновление (только админ)
- `DELETE /api/books/{id}/` - Удаление книги (только админ)
- `GET /api/books/search/` - Поиск книг

**Параметры поиска:**
- `q` - общий поиск
- `title` - по названию
- `genre` - по жанру
- `author` - по автору

## Модели данных

### Книга (Book)
- Название, автор, год выпуска, жанр, категория, издательство
- Изображение обложки, файл книги, описание

### Автор (Author)
- Имя, фамилия, биография, даты рождения и смерти

## Авторизация

Для создания, изменения и удаления книг требуется авторизация администратора (Basic Auth).

## Административный интерфейс

Доступен по адресу: `http://127.0.0.1:8000/admin/`

## Отчет о работе

### Реализованный функционал

1. Модели данных: Author и Book с уникальными ограничениями
2. API эндпоинты для работы с авторами и книгами
3. Система авторизации: Basic Authentication для администраторов
4. Поиск и фильтрация книг и авторов
5. Сериализаторы для преобразования данных в JSON
6. ViewSets для обработки HTTP запросов
